Boa tarde, professor. Tudo certo com o senhor???

  Professor, o meu projeto se trata basicamente de uma loja de carros. Possuímos dois tipos de usuários: o cliente e o administrador, onde tanto o usuário quanto o administrador conseguem visualizar todos os carros disponíveis, fazer buscas dos carros por ano, marca e faixa de preço lá na barra de pesquisa. Porém, para efetuar uma compra, é obrigatório você possuir uma conta e realizar o login. Isso vale tanto para os clientes quanto para os administradores.Na questão de realizar a compra, fiz um sistema completo para tal ato, onde temos 3 etapas. Na etapa 1, é solicitado que você coloque todas as suas informações pessoais; na segunda etapa, você deve colocar as informações da sua residência; e, pra finalizar, na última etapa, você deve escolher a forma em que você vai pagar: parcelamento (tem várias opções de parcelas, calculando o preço final com todos os juros inclusos), à vista ou financiamento.Temos uma diferença de cliente para admin. O admin possuí um botão de painel de controle, onde ele consegue gerenciar todo o site. É ele que consegue adicionar novos carros e suas informações, novos acessórios para os carros, ter controle dos pedidos efetuados, controle dos usuários registrados, além de conseguir gerar um relatório de venda de carros por marca e os 5 carros mais vendidos.
  Toda a criação do projeto foi muito importante. Aprendi comandos do git no terminal, consegui entender a estrutura do projeto, entendi melhor as estruturas de pastas, de como elas são distribuídas, como ver e deixar o código mais limpo. Aprendi muito sobre o banco de dados também, como utilizar o PgAdmin4, como conectar o projeto ao banco de dados para que todas as informações sejam registradas.
  Tive bastante dificuldade em algumas partes, mas a que se destacou foi na questão de finalizar o pedido e o CRUD de pedidos. Após finalizar a compra, aparecia o seguinte: “Erro interno no servidor”. Isso me tirou umas boas horas até que fosse resolvido, pois não era possível finalizar a compra, que consequentemente fazia com que o pedido não fosse enviado para o CRUD pedidos, ou seja, deixava o CRUD totalmente inútil. E, após resolver essa parte onde o pedido era realizado e a compra era totalmente efetuada, apareceu outro erro, que também envolvia o CRUD pedidos. Agora, o pedido não aparecia no seu devido CRUD, e foi isso que mais demorei pra resolver. Tive que utilizar IAs para me ajudar e ter muita paciência, mas, após algum tempo, tudo se normalizou e estava funcionando perfeitamente.
  Esperava ter uma certa dificuldade para concluir o projeto final, porém eu já tinha implementado no terceiro bimestre a grande maioria das novas funcionalidades que foram solicitadas, o que facilitou muitoooo para finalizar completamente o projeto. A única coisa que realmente me atrapalhou foi meu notebook com problema, que me impossibilitou a apresentação do projeto no dia correto.
  Consegui entregar tudo que foi solicitado: visão do cliente e gerente, atualizações no banco de dados, login com credenciais mantidas por cookies, menu principal com o nome do usuário logado e opção de logout, 1 CRUD sem dependências, 1 CRUD com relacionamento 1:n, 1 CRUD para relacionamento n:m e 1 CRUD com relacionamento 1:1, divisão do servidor de DAOs em vários arquivos em suas respectivas pastas (server, router e controller). No frontend, cada CRUD com uma pasta (HTML, JS e CSS).
  Em relação ao uso de IA, foi uma experiência muito boa e traumática ao mesmo tempo, porque elas dão conta de muita coisa, elas sabem resolver os problemas que são solicitados. Porém, quando aparece um problema mais “complexo”, elas tentam resolver ele e acabam mexendo em outras partes do projeto, assim fazendo com que o problema se multiplique. Então, temos que pedir corretamente para que elas realmente entendam o problema, e também ter muita paciência e saber utilizá-las. Utilizei várias IAs diferentes, como o ChatGPT, Gemini, GitHub Copilot, Claude, Manus, entre outras. Mas a IA que mais me auxiliou em todo o projeto foi o Manus. Ele resolvia exatamente o que eu pedia, totalmente ao contrário do ChatGPT, que ficava tentando resolver o mesmo erro muitas vezes e, eventualmente, acabava criando outros problemas. A parte ruim do Manus é que utiliza um sistema de pontos. Quando você cria uma conta, ele te dá 1.300 pontos pra utilizar, mas, quando esses pontos acabam, você ganha apenas 300 pontos diários que não se acumulam, e esses 300 pontos não são o suficiente para resolver a maioria dos problemas. Graças a isso, criei mais de 20 contas no Google para obter pontos e continuar desenvolvendo o projeto. Mas, igual eu disse, foi a IA que mais me ajudou em tudo, então é importante que você escolha a IA que realmente atenda os seus requisitos, isso vai tornar tudo muito mais fácil e menos traumático.
  Gostei muito do frontend e do CSS em geral, porque é muito legal você conseguir customizar seu projeto da forma que você mais se sente confortável, deixar realmente da sua forma. Porém, nem se compara ao backend. Depois que você entende bem do seu backend, das funções que você utilizou e todo o algoritmo do projeto, se torna muito mais interessante e cativante você mexer com os processos internos que ocorrem por trás do projeto. Então, para mim, o backend se tornou bem mais superior que o frontend!!!
