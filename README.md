Professor o meu projeto se trata basicamente de uma loja de carros, possuímos dois tipos de usuários, o cliente e o aministrador, onde tanto o usuario e o administrador conseguem vizualizar todos os carros disponíveis, fazer buscas dos carros por ano, marca e faixa de preço la na barra de pesquisa, porém para efetuar uma compra é obrigatório você possuir uma conta e realizar o login, isso vale tanto para os clientes e os administradores, na questão de realizar a compra, fiz um sistema completo para tal ato, onde temos 3 etapas, na etapa 1 é solicitado que você coloque todas as suas informações pessoais, na segunda etapa você deve colocar as informações da sua residencia, e pra finalizar, na ultima etapa você deve escolher a forma em que você vai pagar, parcelamento(tem várias opções de parcelas, calculando o preço final com todos os juros inclusos), a vista ou financiamento. Temos uma diferença de cliente para admin, o admin possuí um botão de painel de controle, onde ele consegue gerenciar todo o site, é ele que consegue adicionar novos carros e suas informações, novos acessórios para os carros, ter controle dos pedidos efetuados, controle dos usuários registrados, além de conseguir gerar um relatório de venda de carros por marca, e os 5 carros mais vendidos. Toda a criação do projeto foi muito importante, aprendi comandos do git no terminal, consegui entender a estrutura do projeto, entendi melhor as estruturas de pastas, de como elas são distribuídas, como ver e deixar o código mais limpo, aprendi muito sobre o banco de dados também, como utilizar o PgAdmin4, como conectar o projeto ao banco de dados para que todas as informações sejam registradas. 
  Tive bastante dificuldade em algumas partes, mas a que se destacou foi na questão de finalizar o pedido, e o crud de pedidos, após finalizar a compra aparecia o seguinte: "Erro interno no servidor", isso me tirou umas boas horas até que fosse resolvido, pois não era possível finalizar a compra, que consequentemente fazia com que o pedido não fosse enviado para o Crud pedidos, ou seja, deixava o crud totalmente inútil, e após resolver essa parte onde o pedido era realizado e a compra era totalmente efetuada apareceu outro erro, que também envolvia o Crud pedidos, agora o pedido não aparecia no seu devido crud, e foi isso que mais demorei pra resolver, tive que utilizar IAs para me ajudar e ter muita paciência, mas após algum tempo tudo se normalizou e estava funcionando perfeitamente.
  Esperava ter uma certa dificuldade para concluir o projeto final, porém eu ja tinha implementado no terceiro bimestrea a grande maioria das novas funcionalidades que foram solicitadas, o que facilitou muitoooo para finalizar completamente o projeto, a unica coisa que realmente me atrapalhou foi meu notebook com problema, que me impossibilitou a apresentação do projeto no dia correto. Consegui entregar tudo que foi solicitado, Visão do cliente e gerente, atualizações no banco de dados, login com credenciais mantidas por cookies, menu principal com o nome do usuario logado e opção de logout, 1 crud sem dependências, 1 crud com relacionamento 1:n, 1 crud para relacionamento n:m e 1 crud com relacionamento 1:1, divisão do servidor de daos em vários arquivos em suas respectivas pastas(sever, router e controller), no frontend, cada crud com uma pasta(HTML, JS E CSS).
  Em relaçao ao uso de IA, foi uma experiência muito boa e traumática ao mesmo tempo, porque elas dão conta de muita coisa, elas sabem resolver os problemas que são solicitados, porém quando aparece um problema mais "complexo", elas tentam resolver ele e acabam mexendo em outras partes do projeto, assim fazendo com que o problema se multiplique, então temos que pedir corretamente para que elas realmente entendam o problema, e também ter muita paciência e saber para utilizá-las. Utilizei várias IAs diferentes, como o ChatGPT, Gemini, Github Copilot, Claude, Manus, e entre outas, mas a IA que mais me auxiliou em todo o projeto foi o Manus, ele resolvia exatamente o que eu pedia, totalmente ao contrário do ChatGPT que ficava tentando resolver o mesmo erro muitas vezes, e eventualmente acabava criando outros problemas.
